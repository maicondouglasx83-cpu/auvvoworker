# 01-05: Hardening do .htaccess

## Goal
Fortalecer o `.htaccess` com protecoes que faltam: bloqueio de acesso a `.env`, arquivos `*.log`, listagem de diretorios, headers de seguranca HTTP, e redirect HTTP→HTTPS.

## MUST verify (blocking)
- [ ] `.env` nao pode ser acessado via URL
- [ ] Arquivos `*.log` nao podem ser acessados via URL
- [ ] Listagem de diretorios desabilitada
- [ ] Headers de seguranca presentes nas respostas HTTP

## Tasks

### T01: Adicionar bloqueios de acesso a arquivos sensiveis

<read_first>
- `.htaccess` — ler arquivo inteiro (21 linhas)
</read_first>

<action>
Adicionar ao `.htaccess` (antes das regras de Rewrite):

```apache
# Bloquear acesso a arquivos sensiveis
<FilesMatch "\.(env|log|sql|md|json|lock)$">
    <IfModule mod_authz_core.c>
        Require all denied
    </IfModule>
    <IfModule !mod_authz_core.c>
        Order deny,allow
        Deny from all
    </IfModule>
</FilesMatch>

# Impedir listagem de diretorios
Options -Indexes

# Bloquear acesso a arquivos ocultos (exceto .well-known)
<IfModule mod_rewrite.c>
    RewriteRule "(^|/)\.(?!well-known)" - [F]
</IfModule>
```

Notas:
- `.json` e bloqueado porque `design.json` nao deve ser acessivel publicamente
- `.md` bloqueia documentacao interna
- `.sql` bloqueia dumps acidentais
</action>

<acceptance_criteria>
- `grep -c "FilesMatch.*env" .htaccess` >= 1
- `grep -c "Options -Indexes" .htaccess` >= 1
- `grep -c "\.(?!well-known)" .htaccess` >= 1
- Request `GET /.env` retorna 403 Forbidden
- Request `GET /backend/webhook_debug.log` retorna 403 Forbidden
</acceptance_criteria>

### T02: Adicionar headers de seguranca HTTP

<read_first>
- `.htaccess` — ver onde adicionar headers (apos mod_headers ativo)
</read_first>

<action>
Adicionar ao `.htaccess`:

```apache
# Headers de seguranca
<IfModule mod_headers.c>
    # Prevenir clickjacking
    Header always set X-Frame-Options "SAMEORIGIN"
    
    # Prevenir MIME sniffing
    Header always set X-Content-Type-Options "nosniff"
    
    # Ativar protecao XSS do navegador
    Header always set X-XSS-Protection "1; mode=block"
    
    # Politica de referrer (cross-origin reduz informacao)
    Header always set Referrer-Policy "strict-origin-when-cross-origin"
    
    # Remover header de versao do PHP
    Header always unset X-Powered-By
    
    # HSTS (apenas em producao com HTTPS valido — descomentar quando HTTPS confirmado)
    # Header always set Strict-Transport-Security "max-age=31536000; includeSubDomains"
</IfModule>
```

O HSTS fica comentado ate confirmar que HTTPS esta funcionando corretamente.
</action>

<acceptance_criteria>
- `curl -I http://localhost/ 2>&1 | grep -i "x-frame-options"` retorna SAMEORIGIN
- `curl -I http://localhost/ 2>&1 | grep -i "x-content-type-options"` retorna nosniff
- `curl -I http://localhost/ 2>&1 | grep -i "x-powered-by"` retorna vazio ou nao existe
</acceptance_criteria>
