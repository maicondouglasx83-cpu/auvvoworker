# 01-04: Hardening de Sessao

## Goal
Corrigir vulnerabilidades de sessao: session fixation (sem regenerate no login), sem timeout de inatividade, sem SameSite nos cookies, CSRF token nunca rotacionado.

## MUST verify (blocking)
- [ ] Sessao regenerada apos login bem-sucedido
- [ ] Sessao expira apos periodo de inatividade
- [ ] Cookie de sessao com SameSite=Strict
- [ ] Cookie de sessao com Secure=true em producao (HTTPS)

## Tasks

### T01: Adicionar session_regenerate_id no login

<read_first>
- `login.php` — linha ~52 onde login bem-sucedido redireciona
</read_first>

<action>
No `login.php`, apos `$_SESSION['user_id'] = $u['id'];` (linha ~50), adicionar:
```php
// Prevenir session fixation
session_regenerate_id(true);
```

O `true` garante que a sessao antiga e destruida. Isso faz com que um atacante que definiu o session ID da vitima nao consiga reusa-lo apos o login.
</action>

<acceptance_criteria>
- `grep -c "session_regenerate_id" login.php` >= 1
- `grep "session_regenerate_id.*true" login.php` existe
</acceptance_criteria>

### T02: Adicionar timeout de sessao

<read_first>
- `includes/auth.php` — verificar logica atual de verificacao de autenticacao
</read_first>

<action>
Em `includes/auth.php`, modificar a verificacao de autenticacao para incluir timeout:

```php
// Timeout de sessao: 8 horas de inatividade
$sessionTimeout = 8 * 60 * 60; // 8 horas em segundos
if (isset($_SESSION['last_activity']) && (time() - $_SESSION['last_activity'] > $sessionTimeout)) {
    session_unset();
    session_destroy();
    header('Location: /login?expired=1');
    exit;
}
$_SESSION['last_activity'] = time();
```

Adicionar este codigo ANTES da verificacao `if (empty($_SESSION['user_id']))`.

Nota: 8 horas e razoavel para um dia de trabalho. Ajustar conforme necessidade.
</action>

<acceptance_criteria>
- `includes/auth.php` contem `$_SESSION['last_activity']`
- `includes/auth.php` contem verificacao de timeout com `time() - $_SESSION['last_activity']`
- Sessao inativa por > 8h redireciona para /login?expired=1
</acceptance_criteria>

### T03: Configurar cookie de sessao com SameSite e Secure

<read_first>
- `includes/auth.php` — linha `session_start()`
</read_first>

<action>
Em `includes/auth.php`, ANTES de `session_start()`, adicionar:

```php
$cookieParams = [
    'lifetime' => 0,
    'path' => '/',
    'domain' => '',
    'secure' => (!empty($_SERVER['HTTPS']) && $_SERVER['HTTPS'] !== 'off') || ($_SERVER['SERVER_PORT'] ?? '') == 443,
    'httponly' => true,
    'samesite' => 'Strict',
];

// Forcar secure em producao (APP_ENV nao definido = dev)
if (defined('APP_ENV') && APP_ENV === 'production') {
    $cookieParams['secure'] = true;
}

session_set_cookie_params($cookieParams);
```

Mesma configuracao deve ser aplicada em `backend/api.php` (linha ~8) e `login.php` (linha ~3) se usarem `session_start()` independente.
</action>

<acceptance_criteria>
- `includes/auth.php` contem `session_set_cookie_params`
- Parametros incluem `'samesite' => 'Strict'` e `'httponly' => true`
- `secure` e `true` quando `APP_ENV === 'production'`
</acceptance_criteria>

### T04: Rotacionar CSRF token periodicamente

<read_first>
- `includes/auth.php` — funcao de geracao de CSRF token
</read_first>

<action>
Modificar a geracao de CSRF token em `includes/auth.php` para rotacionar a cada 30 minutos:

```php
// Gerar ou rotacionar CSRF token (rotaciona a cada 30 min)
if (empty($_SESSION['csrf_token']) || empty($_SESSION['csrf_token_created']) || (time() - $_SESSION['csrf_token_created'] > 1800)) {
    $_SESSION['csrf_token'] = bin2hex(random_bytes(32));
    $_SESSION['csrf_token_created'] = time();
}
```

Atualmente o token e gerado apenas uma vez por sessao e nunca rotaciona (linhas ~27-29).
</action>

<acceptance_criteria>
- `includes/auth.php` contem `csrf_token_created`
- `includes/auth.php` contem `random_bytes`
- Token rotaciona apos 30 minutos (1800 segundos)
</acceptance_criteria>
