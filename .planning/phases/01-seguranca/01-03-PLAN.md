# 01-03: Webhook Evolution sem Autenticacao

## Goal
Adicionar verificacao de autenticacao no webhook `backend/webhook_evolution.php` para que eventos forjados sejam rejeitados. Atualmente qualquer pessoa que descubra a URL pode disparar eventos falsos.

## MUST verify (blocking)
- [ ] Webhook rejeita requests sem o secret correto (retorna 403)
- [ ] Webhook continua processando requests validos normalmente
- [ ] Evolution API configurada para enviar o header/parametro de autenticacao

## Tasks

### T01: Adicionar verificacao de WEBHOOK_SECRET no webhook

<read_first>
- `backend/webhook_evolution.php` — linhas 22-41 (inicio do webhook, onde nao ha auth)
- `.env.example` — verificar variavel `WEBHOOK_SECRET`
</read_first>

<action>
No inicio de `backend/webhook_evolution.php`, ANTES de qualquer processamento de payload (linha 28+), adicionar:

```php
// Verificar autenticacao do webhook
$webhookSecret = $_ENV['WEBHOOK_SECRET'] ?? null;
if ($webhookSecret) {
    // Evolution pode enviar via header X-Webhook-Secret ou query param ?secret=
    $providedSecret = $_SERVER['HTTP_X_WEBHOOK_SECRET'] 
        ?? $_SERVER['HTTP_X_EVOLUTION_WEBHOOK_SECRET'] 
        ?? $_GET['secret'] 
        ?? null;
    
    if (!$providedSecret || !hash_equals($webhookSecret, $providedSecret)) {
        http_response_code(403);
        echo json_encode(['error' => 'unauthorized', 'message' => 'Invalid webhook secret']);
        exit;
    }
}
```

Notas:
- Usar `hash_equals()` para comparacao timing-safe
- Suportar header customizado (Evolution API permite configurar headers customizados na instancia) e query param como fallback
- Se `WEBHOOK_SECRET` nao estiver definido no .env, logar aviso mas NAO rejeitar (compatibilidade)
- Adicionar log em dev mode do resultado da autenticacao
</action>

<acceptance_criteria>
- `grep -c "WEBHOOK_SECRET" backend/webhook_evolution.php` >= 1
- `grep -c "hash_equals" backend/webhook_evolution.php` >= 1
- Requisicao POST sem header/query secret retorna HTTP 403
- Requisicao POST com secret correto processa normalmente (HTTP 200)
- `WEBHOOK_SECRET` diferente de `EVOLUTION_API_KEY` no `.env.example`
</acceptance_criteria>
